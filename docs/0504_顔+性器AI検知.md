## 1. 概要

顔パイプライン（SCRFD + 周波数解析）と性器パイプライン（Fine-tuned GroundingDino + 周波数解析）を**独立して並列実行**し、最終的に統合判定を行うシステム。

各パイプラインが独立しているため、片方のみ実行することも可能。

## 3. 処理フロー

### 3.1 フレーム抽出（共有ステップ）

| 条件 | 抽出レート |
|------|-----------|
| 動画 < 5分 | 2 fps |
| 動画 ≥ 5分 | 1 fps |

抽出フレームは両パイプラインで共有する。

### 3.2 顔パイプライン

1. **検出**: SCRFD (InsightFace) → 顔 BBox
2. **クロップ**: BBox を 1.2 倍に拡張してクロップ
3. **周波数解析**: Laplacian 分散 + ACF 自己相関
4. **判定**: 閾値未満ならモザイクあり（PASS）、超えていれば漏れ（FAIL）

### 3.3 性器パイプライン

1. **検出**: Fine-tuned GroundingDino → vagina/penis BBox
2. **クロップ**: 検出 BBox をクロップ
3. **周波数解析**: Laplacian 分散 + ACF 自己相関
4. **判定**: 閾値判定で PASS/REVIEW/FAIL

## 5. 統合判定ロジック

```python
VERDICT_RANK = {"PASS": 0, "REVIEW": 1, "FAIL": 2}

def merge_verdicts(face_verdict, genital_verdict):
    """FAIL > REVIEW > PASS（最も深刻な判定を採用）"""
    if VERDICT_RANK[face_verdict] >= VERDICT_RANK[genital_verdict]:
        return face_verdict
    return genital_verdict
```

| 顔 | 性器 | **統合** |
|----|------|----------|
| PASS | PASS | **PASS** |
| PASS | FAIL | **FAIL** |
| FAIL | PASS | **FAIL** |
| REVIEW | FAIL | **FAIL** |
| REVIEW | REVIEW | **REVIEW** |
| ERROR | — | **REVIEW** (ERROR は REVIEW として扱う) |

## 6. 評価結果

### 6.1 性器パイプライン — testset_2603 (2026-05-03)

**評価条件**:
- チェックポイント: `dino_260430_checkpoint0002.pth` (epoch 2)
- box_threshold: 0.3 / text_threshold: 0.2
- プロンプト: `"vagina . penis ."`
- GT ラベル: `vagina_wide`, `penis`, `anus_insertion`
- 評価対象: 性器 GT を持つフレームのみ (n=2353/2837)
- 指標: BBox ベースの Coverage / Precision / IoU

> **❗ 指標の注意（BBox 予測 vs セグメンテーション GT）**  
> GT は LabelMe セグメンテーションアノテーション（性器形状の輪郭ポリゴン）を塗りつぶしたピクセルマスク。予測は GroundingDino 出力の矩形 BBox 塗りつぶし。  
> 矩形 BBox は GT ポリゴンより常に大きいため、**Precision と IoU は体系的に過小評価**される（正しい検出の BBox 内にも非性器ピクセルが常に包まれるため）。  
> **Coverage（Recall）が最も信頼できる指標**。モデル間の相対順位は有効。

| サブセット | n (GT frames) | Coverage | Precision | IoU |
|-----------|:---:|:---:|:---:|:---:|
| normal | 835 | 0.187 | 0.368 | 0.150 |
| gay | 238 | 0.124 | 0.280 | 0.115 |
| hukusu | 662 | 0.323 | 0.427 | 0.276 |
| rez | 618 | 0.148 | 0.125 | 0.084 |
| **全体（加重平均）** | **2353** | **0.209** | **0.312** | **0.165** |

### 6.2 チェックポイント比較

| チェックポイント | 学習日 | epoch | Coverage | Precision | IoU | 判定 |
|---|---|:---:|:---:|:---:|:---:|:---:|
| **dino_260430_checkpoint0002** | 2026-04-30 | 2 | **0.209** | **0.312** | **0.165** | ✅ 採用 |
| dino_0331_checkpoint0004 | 2026-03-31 | 4 | 0.175 | 0.248 | 0.136 | — |

0430 トレーニング（lr=1e-5, bbox_loss=5.0, focal_alpha=0.25）が 0331（lr=2e-5, bbox_loss=4.0, focal_alpha=0.2）を全指標で上回る。ハイパーパラメータ改善により epoch 2 でも epoch 4 以上の性能を達成。

### 6.3 顔パイプライン（参考）

- IoU（Baseline）: **0.566** (別途評価済み)
- 検出率: ほぼ 100%（SCRFD は顔に特化した高精度モデル）

### 6.4 分析

**性器パイプラインの課題:**

1. **Coverage が低い (0.209)**: モデルが GT 領域の約 20% しかカバーできていない。epoch 2 の early checkpoint であるため、学習不足が主因。
2. **Precision は中程度 (0.312)**: 検出した領域のうち約 31% が GT と重なる。  
   **要因 2 つ**: ① 実際の誤検出（体の他部位や背景を性器と誤認） ② **構造的な過小評価**（平屌 BBox 内に必ず GT ポリゴン外の非性器ピクセルが包まれ FP化する）。Precision の理論上限は約 70〜80%。  
   なお、実際の誤検出については FP 分析結果（別項）で詳細を確認すること。
3. **サブセット間の差**: `hukusu`（複数人）が最も高く、`rez`（レズ）が最も低い。学習データのバランスの影響と推測。
4. **box_threshold = 0.3 の影響**: モデルの出力 logit が 0.1〜0.32 程度と全体的に低い。epoch 2 ではまだ確信度が上がっていない。

**改善方針:**
- より多い epoch まで学習したチェックポイントの使用（epoch 10+ を推奨）
- box_threshold を 0.15 に下げることで recall 向上が期待できるが、precision は悪化する
- 追加学習データ（特に rez カテゴリ）

## 8. 既知の制約と今後の改善

### 8.1 現状の制約

| # | 制約 | 影響 |
|---|------|------|
| 1 | チェックポイントが epoch 2 (初期) | logit スコアが低い → 検出漏れ |
| 2 | プロンプトが `vagina . penis .` のみ | anus 検出は間接的にしかカバーされない |
| 3 | BBox ベースの予測マスク | セグメンテーションより粗い |
| 4 | 推論速度 ~110ms/frame | 顔の 5ms と比較して 20 倍遅い |

### 8.2 改善ロードマップ

1. **Phase 1**: epoch 10+ のチェックポイントで再評価 → Coverage 0.4+ を目標
2. **Phase 2**: SAM2 によるセグメンテーションマスクの併用 → IoU 改善
3. **Phase 3**: プロンプト拡張（`"vagina . penis . anus ."`）
4. **Phase 4**: バッチ推論による高速化
