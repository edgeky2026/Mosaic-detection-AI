## 0. 評価指標の構造的注意点

> GT は LabelMe セグメンテーションアノテーション（顔/性器輪郭の機能的ポリゴン）を塗りつぶしたピクセルマスク。予測は矩形 BBox 塗りつぶしマスク。  
> 矩形 BBox は GT ポリゴンより常に大きいため、**Precision と IoU は体系的に過小評価**される（理論上限 Precision≨80%）。  
> **Coverage（Recall）が最も信頼できる指標**。モデル間の相対順位は有効。

## 3. 改善方針

### 3.1 顔: 施策D（SCRFD-34g ハイブリッド）のデプロイ [F2]

**方針**: `config.py` の `use_scrfd34g_hybrid` を `True` に変更し、顔パイプラインのデフォルト動作として 34g ハイブリッドを有効化する。

**期待効果**:
- IoU: 0.529 → 0.579（+9.5%）
- Precision: 0.629 → 0.684+（34g の高精度が寄与）
- Coverage: 34g 単体では -0.01 だが、ハイブリッドで 2.5g にフォールバックするため維持

**実装内容**:
```python
# config.py
use_scrfd34g_hybrid: bool = True  # 施策D をデフォルト有効化
```

### 3.2 顔: conf_th の本番設定 [F1]

**方針**: アップロード時検知（①）用の設定として `scrfd_conf_th=0.50` を用意し、FP 極小化を実現する。

**実装**: 現時点では評価用に 0.3 を維持するが、**本番デプロイ時は 0.50 に切り替える** ことを config.py のコメントに明記。評価スクリプトでは両方の conf_th で結果を出力する。

### 3.3 性器: NMS 重複除去による FP 削減 [G1]

**方針**: GroundingDino の検出結果に対して、IoU ベースの NMS を追加適用し、重複検出を削減する。現行は GroundingDino 内部の NMS のみ依存しており、体部位の大面積 FP が残存する。

**実装**: `genital_pipeline.py` に面積フィルタを追加。フレーム面積の 30% を超える巨大検出は FP として棄却する。

### 3.4 性器: 最小面積フィルタによる FP 抑制 [G1]

**方針**: 検出 bbox がフレーム面積の 0.1% 未満（微小検出）の場合も棄却する。現行は 0.1% だが、追加で**最大面積制限**を設ける。

### 4.1 顔パイプライン改善

**実施内容**:

### 4.2 性器パイプライン改善

**実施内容**:
1. `genital_pipeline.py`: `MAX_BOX_AREA_RATIO = 0.30` の最大面積フィルタ追加
2. `eval_genital.py`: 同一フィルタを追加

**評価結果** (testset_2603, FT model ep4_best, box_thr=0.20, n=2353):

| 設定 | Coverage | Precision | IoU |

**注記**: MAX_BOX_AREA_RATIO=0.30 では効果なし。testset_2603 上のFP検出ボックスは全てフレーム面積の 30% 未満。体部位FPは中小サイズ(1-10%)のため、面積フィルタのみでは除去不可能。

## 5. 改善前後の比較

### 5.1 顔パイプライン

| 指標 | baseline → 施策D | 変化 |
|---|---|---|
| Coverage | 0.6499 → 0.6491 | −0.1% (誤差) |
| Precision | 0.7618 → **0.8024** | **+5.3%** |
| IoU | 0.5663 → **0.5792** | **+2.3%** |

**結論**: SCRFD-34g ハイブリッド（施策D）により Precision と IoU が有意に改善。Coverage は維持。本番運用時は conf_th=0.50 で Precision=0.846 を達成するが、Coverage が 0.548 に低下するトレードオフが存在。

### 5.2 性器パイプライン

| 指標 | 現状値 | 評価 |
|---|---|---|
| Coverage | 0.484 | 改善余地あり（normal=0.321 が特に低い） |
| Precision | 0.480 | FP 体部位問題は面積フィルタでは解決不可 |
| IoU | 0.326 | — |

**未解決課題 (G1)**:
- FP の主因は体部位（腕/脚/腹部）の中小サイズ誤検出
- 面積フィルタ（30%）は安全ガードとして残すが実効性なし
- 今後の改善案: ① 追加学習データ拡充（body part negative mining）、② score threshold 最適化（ただし 0.25→Precision+0.4% のみで Coverage−16%）、③ 後段分類器（検出→肌色/モザイク判別）の追加
