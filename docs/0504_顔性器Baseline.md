# 顔・性器モザイク検知AI — システム設計ベースライン

> 作成日: 2026-05-04  
> 目的: モザイク漏れ検知AIの全体アーキテクチャと各ステージの詳細設計をまとめ、システムの全容を1ドキュメントで把握できるようにする

---

## 1. システム全体アーキテクチャ

### 1.1 設計思想

本システムは「**顔/性器検出器が検出できる ＝ モザイクが不十分**」という逆説的な前提に基づく。十分にモザイク処理された顔/性器は、特徴が破壊されるため検出器に反応しない。検出器が反応した領域に対して周波数解析でモザイクの有無・質を判定する。

## 2. Stage 1: フレーム抽出

### 2.1 仕様

| パラメータ | 値 | 根拠 |
|---|---|---|
| fps_normal | 1.0 fps | 60分未満の動画。1秒1フレームで十分なサンプリング |
| fps_long | 0.5 fps | 60分以上の動画。処理時間抑制 |
| long_video_threshold | 60 分 | 長尺動画の判定閾値 |
| max_resolution | 1920 px | 長辺が超える場合リサイズ |
| jpeg_quality | 95 | DCT周期性への影響を抑制（q≥90で誤差<0.07） |
| scene_change_threshold | 0.3 | シーン変化検出用（将来用） |

### 2.2 実装

```bash
ffmpeg -i input.mp4 -vf "fps=1" -q:v 2 -start_number 0 output/%06d.jpg
```

- 顔パイプラインと性器パイプラインでフレーム抽出は**共有**（1回のみ実行）
- 一時ディレクトリに JPEG 連番として出力

## 3. Stage 2: 検出モデル（事前学習モデル・ファインチューニング）

### 3.1 顔検出: SCRFD

| パラメータ | 値 |
|---|---|
| モデル | SCRFD-2.5g (ONNX, 3.3 MB) |
| 補助モデル | SCRFD-34g (ONNX, 39 MB) — ハイブリッド時のみ |
| conf_threshold | 0.3（評価用） / 0.5（本番想定） |
| nms_threshold | 0.65 |
| max_detections | 15 |
| 入力サイズ | 640×640（内部リサイズ） |
| 出力 | BBox + confidence + 5 keypoints |
| ランタイム | ONNX Runtime (CUDAExecutionProvider) |
| ファインチューニング | **なし**（設計原理上、汎用検出器のまま使用） |

**ハイブリッド運用（施策D）**: 1フレームに1顔 → 34g使用、2顔以上 → 2.5g使用。34g は高精度だが複数顔で2人目を抑圧する傾向があるためフォールバックする。

### 3.2 性器検出: GroundingDINO (Fine-tuned)

| パラメータ | 値 |
|---|---|
| バックボーン | Swin-Transformer Tiny |
| テキストエンコーダ | bert-base-uncased（凍結） |
| テキストプロンプト | `"vagina . penis ."` |
| box_threshold | 0.20 |
| text_threshold | 0.15 |
| モデルサイズ | 1.2 GB (PyTorch) |
| チェックポイント | `dino_local_ft_ep4_best.pth` |
| 出力 | BBox (xyxy) + logit score + phrase |
| ランタイム | PyTorch (CUDA必須) |
| ByteTrack | **なし**（フレーム独立処理） |
| MIN_BOX_AREA | フレーム面積の 0.1% 未満を棄却 |
| MAX_BOX_AREA | フレーム面積の 30% 超を棄却 |

### 3.3 ファインチューニング（性器のみ）

| 項目 | 詳細 |
|---|---|
| 必要性 | 汎用モデルでは Coverage≈0.1 未満で実用不可 |
| 学習データ | finetune_dataset_20260227（15,074枚 train / 2,353枚 val） |
| アノテーション変換 | LabelMe polygon → 外接矩形 → ODVG (.jsonl) |
| ラベル | penis (ID=0), vagina (ID=1) |
| 学習率 | 5e-6, early stop at epoch 8, best = epoch 4 |
| 効果 | Coverage +131%, Precision +54%, IoU +98%（vs GCP baseline） |

## 4. Stage 3: モザイクスコアの算出（周波数解析 3指標）

検出された BBox から ROI を切り出し、以下の3指標で「モザイクの有無・質」を定量評価する。

### 4.1 Laplacian 分散（主指標）

| 項目 | 説明 |
|---|---|
| 算出方法 | ROI → グレースケール → Laplacian フィルタ → 分散値 |
| 意味 | 高周波エネルギー量。高い＝エッジ豊富（モザイクなし or 粗いブロック境界） |
| 顔用閾値 | low=50, high=500 |
| 性器用閾値 | low=15, high=500（性器はテクスチャが少ないため低めに設定） |

### 4.2 ブロックサイズ（ACF: 自己相関関数）

| 項目 | 説明 |
|---|---|
| 算出方法 | ROI → ACF計算 → 第一ピーク位置 = ブロックサイズ推定値 |
| 意味 | ピクセルモザイクの粒度（大きい＝粗いモザイク） |
| 判定基準 | ≥16px → GREEN（十分なモザイク）, 8-15px → YELLOW, <8px → RED候補 |

### 4.3 DCT 周期性

| 項目 | 説明 |
|---|---|
| 算出方法 | ROI → 8×8 DCT → DC成分周辺のピーク比率 |
| 意味 | ブロック境界の周期パターン検出。高い＝モザイクブロック構造が存在 |
| 閾値 | threshold=2.0（超えたらモザイクあり→GREEN寄り）, half=1.5（中間） |

### 4.4 3指標の組み合わせが必要な理由

単一指標では以下の誤判定が発生する：

| ケース | Laplacian単体 | 3指標組み合わせ |
|---|---|---|
| 粗いブロックモザイク (32px+) | 高い（境界エッジ）→ ❌ RED判定 | ACF=32, DCT≥2.0 → ✅ GREEN |
| ガウシアンぼかし | 低い → GREEN | Laplacian<50 → GREEN（正解） |
| 細かいモザイク (4px) | 中程度 → 曖昧 | ACF=4, DCT<2.0 → ✅ YELLOW |

## 5. Stage 4: 3段階スコアリング

### 5.1 顔スコアリング

```
[判定フロー]
1. Laplacian < 50    → GREEN（十分にぼやけている）
2. Laplacian ≥ 500   → RED候補（クリアな高周波）
3. BlockSize ≥ 16    → GREEN（粗いモザイクで十分）
4. Periodicity ≥ 2.0 → GREEN寄り補正（ブロック構造あり）
5. Periodicity < 1.5 → RED寄り補正（周期パターンなし）
6. confidence ≥ 0.85 → RED（信頼度ブースト: 確信度が高い＝モザイク薄い）
7. confidence ≥ 0.70 → YELLOW（中程度の確信度）
8. 上記以外          → 3指標の複合判定
```

### 5.2 性器スコアリング

顔と同じ3指標だが、性器専用閾値を使用：

| パラメータ | 顔 | 性器 | 差分理由 |
|---|:---:|:---:|---|
| Laplacian low | 50 | **15** | 性器はテクスチャ乏しく、低Laplacianでも露出あり |
| Laplacian high | 500 | 500 | 同一 |
| confidence yellow | 0.70 | **0.40** | GDinoスコア帯(0.20-0.64)に合わせて引き下げ |
| confidence red | 0.85 | **0.55** | 同上 |

### 5.3 判定結果の意味

| 判定 | 意味 | 運用上のアクション |
|---|---|---|
| **GREEN** | モザイクが十分に効いている | 問題なし |
| **YELLOW** | モザイクが薄い / 要確認 | 人間レビュー推奨 |
| **RED** | モザイクなし or 極めて薄い | 即時対応が必要 |

## 6. Stage 5: 動画全体の判定（フレーム単位→動画単位への集約）

### 6.1 集約ルール

| 動画判定 | 条件 | 意味 |
|---|---|---|
| **PASS** | RED=0 かつ YELLOW率≤15% | 問題なし |
| **REVIEW** | RED≥1 または YELLOW率>15% | 人間確認が必要 |
| **FAIL** | RED率>5% または 連続RED≥3フレーム | 重大な漏れの疑い |

### 6.2 統合判定（顔＋性器）

```
統合判定 = max(顔_動画判定, 性器_動画判定)
```

重み付け: FAIL > REVIEW > PASS（worst-case wins）。片方が ERROR の場合は REVIEW に繰り上げ。

### 6.3 集約パラメータ

| パラメータ | 値 | 説明 |
|---|---|---|
| red_rate_fail | 0.05 | RED フレームが全体の5%超で FAIL |
| consecutive_red_fail | 3 | 連続3フレーム RED で FAIL |
| yellow_rate_review | 0.15 | YELLOW フレームが15%超で REVIEW |
